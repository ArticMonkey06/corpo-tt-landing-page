# Agregar SECURITY.md

## User Story

Como responsable del proyecto, quiero un archivo `SECURITY.md`, para documentar cómo reportar vulnerabilidades de forma responsable y transmitir profesionalismo y seriedad en seguridad.

## Descripción

El repositorio no tiene política de seguridad. GitHub reconoce `SECURITY.md` (en la raíz o en `.github/`) y lo muestra en la pestaña "Security". Debe incluir: canal de reporte (email de contacto de CorpoTT), versiones/alcance soportado, tiempos de respuesta esperados y qué información incluir en el reporte.

Al ser una landing estática sin backend expuesto, el alcance es acotado, pero conviene igualmente definir el canal de contacto responsable.

## Criterios de aceptación

- Existe `SECURITY.md` (raíz o `.github/`) con: canal de reporte (email de contacto), alcance, tiempo de respuesta estimado y guía de qué incluir.
- El email/canal de contacto fue confirmado con la empresa.
- El archivo está redactado de forma clara y profesional (ES, coherente con el resto del repo).
- GitHub muestra la política en la pestaña "Security".
